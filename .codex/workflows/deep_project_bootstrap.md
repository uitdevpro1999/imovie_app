# Deep Project Bootstrap Workflow

## Purpose

Create a project understanding package before large or risky work.

## Flow

1. Analyst reads project manifests and existing context.
2. Analyst builds architecture, dependency, state, data, route, localization, and test maps.
3. Tech Lead confirms invariants and decision boundaries.
4. Coder uses the package instead of rediscovering the whole project.
5. Reviewer validates against the package and updates it when evidence changes.

## Output

- Deep Package
- Invariant catalog
- Change impact matrix
- Verify ladder
- Architecture risk register
- Confidence level per subsystem
