# Change Logging Workflow

## Purpose

Ensure agent, skill, workflow, command, and context changes are auditable.

## Flow

1. Identify changed files.
2. Explain reason and scope.
3. Record behavior change.
4. Record validation and residual risk.
5. Store log in `context/history/`.

## Rules

- Log the same turn as the change.
- Include blocked validation.
- Keep logs factual and searchable.
