# Technology Library Selection Skill

## Purpose

Choose Flutter/Dart libraries using evidence instead of habit.

## Evaluation Criteria

- Solves the actual requirement
- Fits current architecture
- Maintained and compatible with current SDK constraints
- Minimal transitive risk
- Testable
- Works on target platforms
- Does not duplicate existing capability
- Migration and rollback are clear

## Procedure

1. Check existing dependencies first.
2. Compare built-in Flutter/Dart options against packages.
3. Evaluate candidate package API shape, maintenance, license, platform support, and integration cost.
4. Decide and document tradeoffs.
5. Add dependency only with a clear verify path.

## Output

- Recommendation
- Alternatives considered
- Integration plan
- Verification
- Risks
