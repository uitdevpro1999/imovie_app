# Unstructured Starter Bootstrap Skill

## Purpose

Bootstrap a Flutter starter or incomplete project that has no reliable structure yet.

## Procedure

1. Create a Starter Intake Brief: platforms, feature goal, team skill, deadline, data source, offline needs, auth needs, and test needs.
2. Choose the minimum viable architecture.
3. Define folder structure and naming before generating many files.
4. Create a runnable baseline first.
5. Add one vertical slice that proves routing, state, networking or mock data, and UI composition.

## Rules

- Do not add backend, auth, analytics, flavors, push notifications, or complex CI unless required.
- Keep starter decisions reversible.
- Prefer standard Flutter patterns until scale justifies more infrastructure.

## Output

- Starter Intake Brief
- Starter Architecture Decision
- Starter Blueprint
- Verify Ladder
