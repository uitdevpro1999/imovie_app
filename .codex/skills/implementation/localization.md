# Localization Skill

## Purpose

Keep user-facing text translatable and consistent with the target project's localization system.

## Procedure

1. Detect localization package or custom mechanism.
2. Find resource files, generated keys, and generation commands.
3. Add keys in every supported locale unless the project has a fallback policy.
4. Avoid hardcoded user-facing text in widgets, validators, dialogs, snackbars, errors, and accessibility labels.
5. Run or document localization generation when needed.

## Output

- Keys added/changed
- Locale files impacted
- Generation command
- Missing translation risks
