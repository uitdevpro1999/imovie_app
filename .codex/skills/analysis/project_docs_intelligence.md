# Project Docs Intelligence Skill

## Purpose

Read all provided project documents and convert them into task-ready inputs for agents.

## Procedure

1. Classify each document: product, technical design, API, design, test, release, operational, or unknown.
2. Extract stable facts, volatile assumptions, conflicts, and missing decisions.
3. Build a source registry and traceability matrix.
4. Split work into independent tasks with dependencies, owners, and verification needs.
5. Produce handoff inputs for analyst, tech lead, coder, reviewer, and test writer.

## Output

- Source registry
- Traceability matrix
- Task board
- Dependency map
- Agent input/output contracts
- Open questions
