# Deep Project Understanding Skill

## Purpose

Build a reusable project map for large, risky, or cross-module work.

## When To Use

- New project onboarding
- Large feature
- Architecture change
- Unknown state/data flow
- Repeated defects caused by shallow context

## Deep Package

- Architecture map
- Dependency map
- State lifecycle map
- Data contract map
- Generated/codegen map
- Routing/navigation map
- Localization and asset map
- Test and verification map
- Invariant catalog
- Change impact matrix

## Rules

- Prefer summaries and maps over loading every file.
- Only inspect raw source for ambiguous or high-risk areas.
- Keep findings project-neutral unless the current project proves a convention.
